# 🚗 차량 흠집 감지 프로젝트 — 진행 상황 & 포트폴리오 정리

## 📌 프로젝트 개요

### 한 줄 설명
> "OpenCV 기반 흠집 감지 시스템을 CNN Transfer Learning으로 고도화한 차량 손상 심각도 분류 파이프라인"

### 프로젝트 목표
- before/after 차량 이미지 쌍을 비교해 흠집 여부 및 심각도(0~4단계) 자동 판정
- 기존 OpenCV 파이프라인에 EfficientNet CNN을 결합한 하이브리드 시스템 구축
- FastAPI + HTML/CSS/JS 웹 서비스로 배포

### 심각도 라벨링 기준 (직접 설계)
| 단계 | 수준 | 기준 |
|------|------|------|
| 0 | 정상 | 신규 흠집 없음 |
| 1 | 경미 | 얕은 스크래치, 멀리서 잘 안 보임 |
| 2 | 보통 | 육안으로 명확, 도장 손상, 부분 도색 가능 |
| 3 | 심각 | 넓거나 깊음, 찌그러짐, 수리 필요 |
| 4 | 매우 심각 | 구조적 손상 수준, 즉시 수리/교체 필요 |

## ✅ 완료된 단계

### 1단계 — 데이터 준비 ✅
- **파일**: `step1_prepare_data.py`
- OpenCV 파이프라인으로 before/after diff → ROI 패치 자동 추출
  - ECC 정렬로 카메라 이동 보정
  - Canny 엣지 + Morphology로 흠집 영역 마스크 생성
  - 패치 미감지 시 after 전체 이미지 사용
- 데이터 증강으로 634개 → 4,190개
- train/val 자동 분리 저장

### 2단계 — CNN 학습 ✅
- **파일**: `step2_train_cnn.py`
- 모델: EfficientNet-B0 (ImageNet pretrained) Fine-tuning
- 클래스 불균형 대응: CrossEntropyLoss 가중치 적용
- Early Stopping (patience=5) 적용
- **결과**:
  - val accuracy: **64%**
  - 0-정상 F1: 0.77 ✅
  - 4-매우심각 F1: 0.21 ⚠️ (데이터 부족)
- 모델 저장: `C:\Users\woain\Python_AI\car\model\best_model.pth`

## 💡 포트폴리오 설명 포인트

### 기술적 도전과 해결
1. **데이터 부족 문제**
   - 126쌍 → OpenCV 패치 추출 + 데이터 증강으로 4,190장 확보
   - Transfer Learning으로 적은 데이터에서도 학습 가능하게 설계

2. **클래스 불균형 문제**
   - score 0이 46대로 가장 많고 score 4가 14대로 가장 적음
   - CrossEntropyLoss에 클래스 가중치 적용해 해결

3. **이미지 정렬 문제**
   - before/after 촬영 각도 미세 차이 → ECC 알고리즘으로 정렬 보정

4. **하이브리드 파이프라인 설계**
   - OpenCV: 흠집 위치 감지 (Detection)
   - CNN: 심각도 분류 (Classification)
   - 두 기술의 장점을 결합한 실용적 접근

### 한계 및 개선 방향 (솔직하게)
- val accuracy 64%: 126쌍의 제한된 데이터로 인한 한계
- 추가 공개 데이터셋(CarDD 등) 결합 시 성능 향상 가능
- 실제 서비스에서는 사용자 피드백으로 지속 학습 구조 고려
